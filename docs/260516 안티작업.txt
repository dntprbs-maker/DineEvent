# DineEvent 프로젝트 작업 로그 (2026-05-16)

## [01:00 - 01:45] 모바일 전용 대시보드 신규 개발 및 폐기
- **작업 내용**: 스마트폰 환경에 최적화된 '프리미엄 모바일 대시보드(AdminDashboardMobile.jsx)' 신규 제작.
- **주요 기능**: 하단 탭 시스템(식당/이벤트/고객관리), 요약 카드(그라데이션 디자인), 다크모드 UI.
- **결과**: 사용자 요청에 따라 신규 페이지 방식은 폐기하고 기존의 'MobileAdminMessages.jsx'를 개선하는 방향으로 선회.
- **되돌리기**: 생성했던 파일 삭제 및 App.jsx 라우트 원복 완료.

## [01:50 - 02:10] 기존 모바일 화면 1차 개선 (아기자기한 요약 카드 추가)
- **작업 내용**: 'MobileAdminMessages.jsx'에 신규 대시보드에서 호평받은 요약 카드 섹션 이식.
- **주요 기능**: 실시간 총 응모/오늘 응모/당첨자 통계 카드 추가, 기간 필터링 로직 추가.
- **결과**: 디자인이 너무 화려하다는 피드백에 따라 요약 카드를 제외하고 기능만 남기기로 결정.

## [02:15 - 02:25] 기존 모바일 화면 2차 개선 (기능 집중)
- **작업 내용**: 화면을 원래 상태로 되돌린 후, '기간 필터' 기능만 검색창 라인에 삽입.
- **수정 사항**: 검색창 앞에 달력(📅) 아이콘 버튼 배치 및 검색창 너비 조정.
- **되돌리기**: 기존 디자인을 유지하기 위해 1차 개선에서 추가한 상단 요약 카드 모두 제거.

## [02:30 - 02:35] UI 레이아웃 버그 수정
- **현상**: 검색창의 `fixed` 속성 때문에 달력 버튼이 뒤로 가려져 보이지 않는 문제 발생.
- **해결**: 달력 버튼을 검색창(`entry-search-wrapper`) 내부로 통합하여 레이아웃 충돌 해결. 드롭다운 필터 창도 고정 위치로 조정.

## [02:36 - 02:40] SMS 발송 로직 및 안내 문구 수정
- **현상**: 필터를 걸었음에도 실제 문자 발송 시 전체 고객(`uniqueData`)에게 문자가 가는 심각한 오류 발견.
- **해결**: 필터링된 결과(`filteredEntries`)를 기반으로 발송 대상자를 추출하도록 로직 전면 수정. 모달창 안내 인원수도 정확하게 표시되도록 변경.

## [02:41 - 02:45] 알림 시스템 고도화 (Native Alert 대체)
- **현상**: 일부 모바일 환경에서 대상자가 0명일 때 뜨는 `alert()` 경고창이 작동하지 않는 문제 발생.
- **해결**: 브라우저 기본 경고창 대신, 프로젝트 테마에 맞춘 '프리미엄 경고 모달창'을 신규 제작하여 100% 노출 보장.

---
**최종 상태**: 기존 모바일 관리 화면 디자인을 보존하면서, 강력한 기간 필터링 기능과 안전한 문자 발송 시스템 구축 완료.
